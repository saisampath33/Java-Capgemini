| Feature                    | `compareTo()`                                  | `compare()`                             |
| -------------------------- | ---------------------------------------------- | --------------------------------------- |
| Belongs to                 | `Comparable` interface                         | `Comparator` interface                  |
| Purpose                    | Defines **natural ordering** of a class        | Defines **custom / multiple orderings** |
| Implemented in             | The class itself (e.g., `Student`, `Employee`) | Separate class (or lambda)              |
| Method signature           | `int compareTo(T o)`                           | `int compare(T o1, T o2)`               |
| Number of objects compared | Compares **this** with one object              | Compares **two external objects**       |
| How many sorting rules     | Only ONE                                       | Can have MANY                           |
| Used when                  | Class has a single default sort logic          | You need flexible sorting               |
| Example                    | Sort Employee by ID                            | Sort Book by price / title              |
| Sorting call               | `Collections.sort(list)`                       | `Collections.sort(list, comparator)`    |
| Primitive support          | Needs wrapper (`Integer`, `Double`)            | Usually uses `Integer.compare()` etc    |
| Modifies class?            | YES                                            | NO                                      |
| Real-world analogy         | Default behavior                               | External strategy                       |
