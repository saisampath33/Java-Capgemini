Types of HAS-A Relationship

Two forms of association -> aggregation and composition
Aggregation (Weak HAS-A)

Objects can exist independently - College HAS-A Students (Even if college closes, students still exist) and Coupling - LOOSE by lazy loading



Composition (Strong HAS-A)

Objects cannot exist independently - House HAS-A Rooms (If house is destroyed, rooms are destroyed) and Coupling - TIGHT by using early loading


Reusability - Runtime
Coupling - LOOSE
Better Maintainability

Here, One object depends on other and we can achieve this by creating a reference variable for it

One class contains a reference variable
That reference variable points to an object of another class
One class contains reference variable of another class

Both aggregation and composition use reference variables, but aggregation creates objects outside the class, while composition creates objects inside the class.

Lazy Instantiation

We will only create the object when we need it(later)
Saves memory
Improves performance
Useful for heavy objects (DB, network, files)
Avoids unnecessary object creation.
